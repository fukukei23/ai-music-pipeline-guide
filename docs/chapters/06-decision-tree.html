06 使い分け判断木 — 目的別スキル選択

「何をしたいか」から、どのスキルを使うかが決まる。迷ったらこのチャートで。

目的別チャート

曲を作りたい

新曲を作る        → make-song
テーマ特化（三国志/サイバー和）→ sangoku-song / cyber-wa-song

既存曲を分析したい

「どんな曲か」を言葉で（再現したい）
   → reverse-engineer-song（定性・Gemini）

BPM/キー/コード等を数値で（客観比較）
   → analyze-song（定量・librosa/music21）

💡 両方走らせれば、言語+数値の両面から曲を理解できる。

映像が欲しい

曲（音源）からMV・ショート映像を
   → video-prompt-spec（映像4構造プロンプト）

クイックリファレンス

やりたいこと スキル トリガー例
オリジナル曲を作曲 make-song 「曲作って」
YouTube曲をRE reverse-engineer-song 「この曲から仕様書作って」
曲のBPM/コード抽出 analyze-song 「楽曲分析して」
MVの映像プロンプト video-prompt-spec 「映像プロンプト」

よくある誤選択

⚠ 「映像プロンプトが欲しい」のに reverse-engineer-song を使う: 映像は video-prompt-spec の役割。RE は音楽/画像/動画のプロンプト仕様書だが、映像制作の実務は video-prompt-spec が専門。

⚠ 「曲を分析したい」のに make-song を使う: make-song は作曲専用。分析は RE（定性）か analyze-song（定量）。

出力先の違い

  • make-song: 曲+歌詞（songs/<曲名>/）
  • reverse-engineer-song: プロンプト仕様書（3モダリティ）
  • analyze-song: features.json + 楽譜 + report.md
  • video-prompt-spec: 映像4構造プロンプト仕様書

出力形式が違うので、目的に合ったスキルを選ぶ。
