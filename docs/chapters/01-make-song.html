01 make-song — 汎用作曲エンジン

任意ジャンル・テーマで曲＋歌詞を制作する。曲先（メロディ淘汰型）でキャッチーなサビを確定してから歌詞を載せる、ヒット曲プロセス準拠。

できること

  • 任意のジャンル・テーマでオリジナル曲を制作
  • 曲先方式: サビメロディを複数生成→人間が選別→変奏淘汰で確定
  • 歌詞の載せまで一気通貫
  • 主軸AI: MiniMax music-2.6

トリガー

「曲作って」「作曲して」「音楽作って」「歌作って」「/make-song」

テーマ修飾付き（「武将で曲作って」=sangoku-song、「サイバー和で」=cyber-wa-song）は専用スキルが優先。

曲先プロセス（核心）

ヒット曲は「先にメロディが決まる」。make-song はこのプロセスを再現:

  1. サビメロディを複数生成
  2. 人間が選別（淘汰）
  3. 変奏で更に絞り込み→確定
  4. 確定メロディに歌詞を載せる

メロディ淘汰を経ることで「耳に残るサビ」を確率的に引き当てる。

知見アーキテクチャ（4層×5象限）

AI依存度で知見を4層に分離。AI（MiniMax）が替わっても Layer 2 だけ差し替えれば済む設計。

  • Layer 1: AI不変の音楽理論（コード進行・スケール・構造）
  • Layer 2: AI依存の実測知見（MiniMax の パラメータ・声質・圧縮率）
  • Layer 3: プロンプト技法
  • Layer 4: 実践フロー

詳細はAI作曲エンジン教科書（姉妹サイト）で公開。

他スキルとの連携

  • 上流: reverse-engineer-song / analyze-song の知見を参照して「近い型」で作曲
  • 下流: video-prompt-spec で完成曲を映像化

成果物の保存

~/projects/<リポジトリ>/songs/<曲名>/ 配下。~/Music/ には保存しない（固定ルール）。
